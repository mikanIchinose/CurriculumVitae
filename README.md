## 強み

業務効率化に非常に興味があり、繰り返し実行される作業は極力シェルなどを使ってスクリプト化するようにしています。また何度の書くことについてはテンプレートを作ったり、補完プラグイン用のデータソースを自作したりして書き方がぶれないように仕組み化することを得意としております。
チームや実装の課題を俯瞰的に捉えて、現実的な速度で改善していくことを得意としています。
アーキテクチャについても強い関心を持っており、コンポーネント間の結合や、コンポーネント自身のモジュール性について常に意識しながら設計・実装を行っています。MVVMやクリーンアーキテクチャを用いた開発経験から、保守性の高いコードを書くことを心がけています。
